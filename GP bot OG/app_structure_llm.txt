## Architecture
```
                      ┌─────────────────────────────────────────┐
                      │  Same codebase, differentiated by       │
                      │  BOT_ID env var (NEW_BEST/BALANCED/GOLD_PLUS) │
                      └─────────────────────────────────────────┘

market-data.ts → whale-watcher.ts → filter-engine.ts → risk-manager.ts → executor.ts
     ↑                                                                         ↓
  Binance WS                                                           bot-server.ts
  (+ REST fallback)                                                    (Express + WS)
  Polymarket WS                                                            ↓
  Gamma API                                                        bot-dashboard.html
```

## Dynamic Position Sizing
- NB: High Conviction ($20) when entry >= 0.80, Standard ($10) otherwise
- BAL: Flat $10 always (forced via BAL_LIVE_OVERRIDES — no high-conviction tiers)
- GP: Flat $10 always (OG GOLD: simple and consistent — no high-conviction tiers)
- Sizes configurable via dashboard settings (BAL and GP force $10/$10)

## Position Stacking
- NB: Up to 3 entries per conditionId (configurable maxEntriesPerContract)
- BAL: maxEntriesPerContract = 1 (no stacking — disabled for LIVE safety)
- GP: maxEntriesPerContract = 1 (no stacking — OG GOLD keeps it simple)
- Stack only if whale trade size >= $25 (configurable minStackSize)
- 5-second cooldown between entries on same contract
- Each stacked entry counts individually toward maxOpenPositions

- Max Loss/Hour: $100 (NB), $150 (BAL), $100 (GP) — per-bot overrides in config.ts

- Per-contract cooldown: 5000ms
- BTC + ETH only, BUY side only
- All bots: 5-min + 15-min contracts (GP upgraded from 5-min only in v9)
- BAL dead hours skip: 02-05 UTC
- Excluded wallets (LIVE only): configurable per bot, default 0x37c9 + 0x571c

### BAL LIVE Overrides (conservative, layered on top of shared LIVE_OVERRIDES)
- maxOpenPositions: 10

## LIVE Trading
- CLOB client: Polymarket ClobClient with signatureType=1 (POLY_PROXY)
- Orders: FOK (Fill-Or-Kill) at whale price + 0.02 buffer, feeRateBps=1000
- Auto-fallback: 3 consecutive order failures → automatic PAPER switch
- Wallet: 0xdb67... (shared between all bots — only one LIVE at a time)
- BAL LIVE enabled since v8 (was PAPER-only before)

## CSV File Rotation
- Rotation threshold: 70K lines (changed from 90K in v8)
- Max archives: 10 per dataset
- Rotated datasets: decisions, bot_trades, live_events
- Archives stored in data/archives/ as CSV files with header
- Rotation check: every 60 seconds + on each append

## Latency Chain (whale trade → bot order)
- Polymarket API indexing lag: ~2-5s (uncontrollable)
- Poll interval: 3s per wallet, staggered 750ms (avg 1.5s wait)
- HTTP request: ~200-500ms
- Processing (filter + risk + sizing): ~1-5ms
- LIVE order placement (FOK): ~250-600ms
- **Total typical: ~5-7s** | Worst case: ~10s+
- v8 quick wins: non-blocking REST book fetch, latency breakdown logging (detectedAt, apiLag, process)

## SELL Trade Handling
- Bot only copies BUY trades (SELL excluded by filter)
- Spy server: SELL trades preserved in CSV but won/pnl columns are empty (not computed)
- Bot whale CSV export: won/pnl only computed for BUY trades
- All WR/PnL statistics are BUY-only

## Data Recording (per bot trade)
Each trade records: botId, contractDuration, sizeReason, stackEntry, stackTotal,
stackTriggerSize, whaleTxHash, latencyMs, binancePrice, polyMidAtDecision, bookSpread,
btcVol1h, concurrentWhales, sessionLabel, orderBookDepth, whaleUsdcSize, whalePrice,
btcDelta30s, btcDelta5m, slippage, fillPriceVsMid, confirmed.
Column `whaleTxHash` is consistent across spy, decisions, and bot_trades CSVs for joins.

## Key Design Decisions
- Single codebase, three deployments: BOT_ID env var selects active filter
- midEdge is INVERTED: negative midEdge = whale pays ABOVE mid = high conviction
- Fresh market data at decision time: polyMidAtDecision and bookSpread read from
  tokenBook at OUR evaluation time, not the whale's snapshot (3-7s earlier)
- BTC + ETH only (SOL/XRP = noise per 82K analysis)
- BUY-only (SELL trades from 0x37c9 have no profit)
- 5+15 minute contracts (1-min too fast, hourly too slow)
- Paper mode default — LIVE enabled per bot via dashboard settings
- Auth token required for all API access and WS connections
- Binance WS with REST fallback for Railway EU deployment
- Railway persistent volume at /app/data for trade + settings persistence
- File rotation: 70K lines → CSV archive, keeps 10 archives per dataset
- Non-blocking REST book fetch: fire-and-forget to avoid blocking trade pipeline
- Wallet shared between bots: only one LIVE at a time to avoid position conflicts
